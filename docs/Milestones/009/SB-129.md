- [ ] 10. Verify catch-up stores and forwards the missed note exactly once.
- [ ] 11. Restart `capture-service` again.
- [ ] 12. Verify no duplicate row and no duplicate receipt appear.
- [ ] 13. Post a likely-secret test string (e.g. `password=hunter2`).
- [ ] 14. Verify only a redacted rejection record exists in SQLite.
- [ ] 15. Verify the likely-secret string never reaches n8n (no n8n execution for this message).

## Operations runbook

Document the following in a `docs/OPERATIONS.md` file:

### Service management

- Start all services: `docker compose up -d`
- Stop all services: `docker compose down`
- Rebuild and restart: `docker compose up -d --build`
- View logs: `docker compose logs -f <service>`

### Health checks

- `curl http://localhost:8000/health` — capture-service
- `curl http://localhost:8001/health` — writer-service
- `secondbrain status` — full operational status

### Manual retry

```bash
secondbrain retry SB-YYYYMMDD-NNNN
```

Exit code 0 = queued. Exit code 1 = rejected (not found or not in FAILED state).

### Common failure modes

| Symptom | Likely cause | Resolution |
|---------|-------------|------------|
| No receipt after Discord message | capture-service down or Discord token invalid | Check `docker compose logs capture-service` |
| Receipt stuck at ⏳ | n8n down or webhook misconfigured | Check n8n health, verify webhook URL in n8n workflow |
| Filing failed after classification | writer-service down or Git push rejected | Check `docker compose logs writer-service`; manual retry |
| Stale `.git/index.lock` | writer-service crashed mid-commit | writer-service refuses to start; safe repair required; do not delete lock blindly |
| `secondbrain status` exits 2 | SQLite ledger not found | Verify `LEDGER_PATH` env var and volume mount |

### Accessing n8n

Document the chosen private access method from SB-127.

### Backup and restore

- Backups run nightly via cron.
- Restore test: `deploy/restore-validate.sh`.
- Never run restore against live volumes.
