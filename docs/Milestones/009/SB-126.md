# SB-126 — Deploy V3/M8-hardened system to EC2 with security hardening

**Branch:** `release/production`

---

## Goal

Get the post-M8 V3 system running on EC2 with the security posture defined in Architecture section 9.

---

## EC2 host hardening checklist

- [ ] SSH key-based login confirmed working.
- [ ] Password-based SSH login disabled (`PasswordAuthentication no` in `sshd_config`).
- [ ] EC2 security group allows inbound SSH (port 22) from known IPs only.
- [ ] No inbound rules for port 5678 (n8n), port 8000 (capture-service), or port 8001 (writer-service) on the EC2 security group.
- [ ] Docker and Docker Compose are installed and functional.

## Service deployment checklist

- [ ] Clone or pull the `main` branch onto EC2 under a dedicated directory (e.g. `/opt/second-brain/app`).
- [ ] Create the runtime data directories: `/var/lib/second-brain/` and `/opt/second-brain/vault`.
- [ ] Write production `.env` on EC2 (not committed to git); verify all required variables are set.
- [ ] Run `docker compose up -d --build`.
- [ ] Verify all containers start: `docker compose ps` shows all services as `running` or `healthy`.
- [ ] Confirm `capture-service` health: `curl http://localhost:8000/health` from EC2 host returns 200.
- [ ] Confirm `writer-service` health: `curl http://localhost:8001/health` from EC2 host returns 200.
- [ ] Confirm n8n is reachable through the private access layer (SB-127), not directly via port 5678.

## Credential isolation checklist

- [ ] Discord bot token is present only in the `capture-service` environment.
- [ ] GitHub write credential (deploy key or scoped PAT) is present only in the `writer-service` environment.
- [ ] Gemini API key is present only in n8n environment variables.
- [ ] The SQLite file is mounted only into the `capture-service` container.
- [ ] Git credentials are not mounted into n8n or `capture-service`.

---

## Acceptance criteria

- All three service containers are healthy after `docker compose up -d --build`.
- Port 5678 is not reachable from the public internet.
- No internal service port is published to `0.0.0.0` on the EC2 host.
- SSH password authentication is disabled.
- A single Discord test message produces a saved receipt within a few seconds.

**Done when:** all checklist items are checked and the Discord test message produces a receipt.
