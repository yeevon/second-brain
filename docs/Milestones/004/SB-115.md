# SB-115 — Add GitHub vault sync and serialized Git writes

**Branch:** `feature/writer-git-sync`

Add a Git-backed vault to `writer-service`. Every successful note write is committed and pushed to a private GitHub repository. An OS-level advisory `flock` serializes all writes so concurrent requests never corrupt the repository.

---

## Goal

After SB-115:

```text
POST /internal/notes/file
    ↓
acquire OS advisory flock on /opt/vault/.writer.lock
    ↓
git fetch origin
    ↓
git merge --ff-only origin/main
    ↓
check whether capture_id already exists in vault
    ↓
render Markdown note
    ↓
write file to /opt/vault/<path>
    ↓
append audit event to 99_log/events.ndjson
    ↓
git add <note_path> 99_log/events.ndjson
    ↓
git commit -m "note: <capture_id> via writer-service"
    ↓
git push origin main
    ↓
release flock
    ↓
return { note_path, git_commit_hash }
```

The vault is a private GitHub repository cloned at `/opt/vault` on the EC2 instance. After SB-115, every filed note is visible in the GitHub repository.

---

## Architecture rules

### writer-service is the only Git actor on the vault

No other process commits to or pushes from the EC2-side vault clone. `capture-service`, `n8n`, and supporting scripts must not run `git` commands against the vault path.

### The flock is kernel-managed, not file-existence-based

Use `fcntl.flock(fd, fcntl.LOCK_EX)` on an open file descriptor. The kernel releases the flock automatically when the process terminates or the fd is closed. A file-existence lock remains even after an unexpected crash.

Do not use:

```text
/opt/vault/.writer.lock exists → locked
/opt/vault/.writer.lock absent → free
```

Use:

```text
open("/opt/vault/.writer.lock", "w")
    ↓
fcntl.flock(fd, fcntl.LOCK_EX)  # blocks until lock acquired
    ↓
... write sequence ...
    ↓
fcntl.flock(fd, fcntl.LOCK_UN)
fd.close()
```

### The write sequence is atomic with respect to the flock

Acquire the flock before any Git or filesystem mutation. Release it only after `git push` completes or a failure is detected. Never release the flock before the push is confirmed.

## 4. Implement the advisory flock

Add `src/writerservice/flock.py`:

```python
import contextlib
import fcntl
import pathlib
from typing import Generator

@contextlib.contextmanager
def vault_write_lock(lock_path: pathlib.Path) -> Generator[None, None, None]:
    lock_path.touch(exist_ok=True)
    fd = lock_path.open("w")
    try:
        fcntl.flock(fd, fcntl.LOCK_EX)
        yield
    finally:
        fcntl.flock(fd, fcntl.LOCK_UN)
        fd.close()
```

Use this context manager to wrap every write operation in `writer.py`:

```python
with vault_write_lock(vault_path / ".writer.lock"):
    _perform_git_write(...)
```

The lock file is `.writer.lock` inside the vault root. It is not tracked by Git. Add it to the vault repository `.gitignore`.

Do not use `asyncio.Lock` as the primary write serializer. Advisory flock works across threads and processes; asyncio.Lock does not span process boundaries.

## 5. Implement the Git write sequence

Add `src/writerservice/git_ops.py`. Implement these operations as thin subprocess wrappers:

```python
def git_fetch(vault_path: Path) -> None: ...
def git_merge_ff_only(vault_path: Path) -> None: ...
def git_add(vault_path: Path, *paths: str) -> None: ...
def git_commit(vault_path: Path, message: str) -> None: ...
def git_push(vault_path: Path) -> None: ...
def git_rev_parse_head(vault_path: Path) -> str: ...
def git_log_hash_for_path(vault_path: Path, note_path: Path) -> str | None: ...
```

`git_log_hash_for_path` resolves the commit hash of the most recent commit that touched `note_path`:

```python
def git_log_hash_for_path(vault_path: Path, note_path: Path) -> str | None:
    result = subprocess.run(
        ["git", "log", "-1", "--format=%H", "--", str(note_path.relative_to(vault_path))],
        cwd=vault_path,
        capture_output=True,
        text=True,
    )
    if result.returncode != 0 or not result.stdout.strip():
        return None
    return result.stdout.strip()
```

Used by idempotent replay to return the existing commit hash instead of `None`.

Each function must:

```text
run git as a subprocess inside vault_path
capture stdout and stderr
raise a typed exception on non-zero exit code
not print or log stdout/stderr verbatim (may log safe summary)
not include raw capture text in the commit message
```

### Commit message format

```text
note: <capture_id> via writer-service
```

Never include classification body, raw Discord text, Gemini output, or user-supplied free-form text in the commit message.

### Push return value

After `git push`, resolve the current HEAD commit hash:

```bash
git rev-parse HEAD
```

Return the hash as the `git_commit_hash` in the API response.

## 6. Implement the full write sequence in writer.py

Update `src/writerservice/writer.py`. Replace the local-filesystem-only write with the full sequence:

```python
def file_note(request: FileNoteRequest) -> FileNoteResult:
    with vault_write_lock(config.vault_path / ".writer.lock"):
        git_fetch(config.vault_path)
        git_merge_ff_only(config.vault_path)

        existing = find_note_by_capture_id(config.vault_path, request.capture_id)
        if existing is not None:
            existing_hash = git_log_hash_for_path(config.vault_path, existing)
            return FileNoteResult(
                result="FILED",
                note_path=existing,
                git_commit_hash=existing_hash,
                idempotent=True,
            )

        note_path = derive_note_path(request)
        write_note_file(config.vault_path, note_path, request)
        append_audit_event(config.audit_log_path, request, note_path)

        git_add(config.vault_path, note_path, "99_log/events.ndjson")
        git_commit(config.vault_path, f"note: {request.capture_id} via writer-service")
        commit_hash = git_rev_parse_head(config.vault_path)
        git_push(config.vault_path)

        return FileNoteResult(
            result="FILED",
            note_path=str(note_path),
            git_commit_hash=commit_hash,
            idempotent=False,
        )
```

The function must hold the flock for the entire duration. `git rev-parse HEAD` is called before `git push` so the commit hash is captured from the local commit; this also makes the rollback boundary clear — the hash is known regardless of whether the push succeeds.
