# SB-121 — Add weekly review workflow

**Branch:** `feature/weekly-review`

## Purpose

Run a weekly review on EC2 that reads the capture ledger (and optionally the vault) and posts a structured summary plus an AI-generated priorities section to `#brain-digest` or DM. Completion must be based on explicit state changes only — never inferred from vague prose.

---

## Completion rule

Count only explicit prefix-driven note types from `classification_json`:

- `task:` captures → `created_tasks_count`
- `done:` captures → `completed_actions_count`
- `fix:` corrections → included in `completed_actions_count` and `corrections_count`
- `decision:` captures → `decisions_count`

Natural-language body text is never scanned for completion signals. Vague prose ("I finished the thing") is invisible to this workflow.

---

## Config

```env
WEEKLY_REVIEW_ENABLED=true
WEEKLY_REVIEW_DAY=SUN
WEEKLY_REVIEW_TIME=18:00
DISCORD_DIGEST_WEBHOOK_URL=       # n8n workflow variable ($vars.DISCORD_DIGEST_WEBHOOK_URL)
DIGEST_TIMEZONE=America/Chicago   # configured in n8n schedule trigger timezone field
VAULT_PATH=                       # required for outstanding_tasks_count; degrades gracefully without it
```

`WEEKLY_REVIEW_DAY` and `WEEKLY_REVIEW_TIME` are set in the n8n schedule trigger node, not as runtime env vars. Current workflow fires Monday 8 am UTC — align with target timezone during deployment.
