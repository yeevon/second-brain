# SB-121 — Add weekly review workflow

**Branch:** `feature/weekly-review`

## Purpose

Run a weekly review on EC2 that reads current vault state and posts a structured summary plus an AI-generated priorities section to `#brain-digest` or DM. Completion must be based on explicit state changes only — never inferred from vague prose.

---

## Completion rule

Count only explicit note state from vault frontmatter:

- `note_type: done` / `note_type: fix` notes → accomplished
- actions with `status: done` / `status: completed` → completed tasks
- `note_type: decision` notes → decisions
- open actions → still-open work

Natural-language body text is never scanned for completion signals. Vague prose ("I finished the thing") is invisible to this workflow.

---

## Config

```env
WEEKLY_REVIEW_ENABLED=true
WEEKLY_REVIEW_DAY=SUN
WEEKLY_REVIEW_TIME=18:00
DISCORD_DIGEST_WEBHOOK_URL=       # read by n8n as $env.DISCORD_DIGEST_WEBHOOK_URL
DIGEST_TIMEZONE=America/Chicago   # configured in n8n schedule trigger timezone field
VAULT_PATH=                       # fallback vault scan path if writer-service is unavailable
```

`WEEKLY_REVIEW_DAY` and `WEEKLY_REVIEW_TIME` are set in the n8n schedule trigger node, not as runtime env vars. Current workflow fires Monday 8 am UTC — align with target timezone during deployment. `DISCORD_DIGEST_WEBHOOK_URL` must be available to the n8n container as `$env.DISCORD_DIGEST_WEBHOOK_URL`.
