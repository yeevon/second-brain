# SB-123 — Add the local read-only MCP server

**Branch:** `feature/read-only-mcp`

**Status:** Core implementation done. Remaining work is the Gemini CLI integration (SB-124).

## Purpose

Expose the local vault and capture ledger through a narrow read-only MCP server (`brain-mcp`) so local AI clients (Claude Code, Gemini CLI via gembrain) can query notes and tasks. Runs as a host process — not inside Docker.

---

## Implementation checklist

### MCP server (`src/secondbrain/mcp_server.py`)

- [x] `brain-mcp` entry point in `pyproject.toml`
- [x] STDIO transport (`mcp.server.stdio.stdio_server`)
- [x] `mcp>=1.0.0` dependency

#### Tools

- [x] `search_notes(query, folder?, project?, tags?, limit?)`
- [x] `read_note(note_path)` — relative to vault root
- [x] `list_recent_notes(days?, folder?, limit?)`
- [x] `list_open_tasks(project?, limit?)` — delegates to `scan_open_task_list()` in `digest.py`

#### Safety constraints

- [x] Path-root enforcement — resolves paths against vault root; rejects traversal
- [x] Hidden-path guard in `read_note` — rejects `.`-prefixed path components
- [x] `.md`-only constraint in `read_note`
- [x] Result limit clamped to `[1, 100]`, default 20
- [x] No mutation tools
- [x] No shell execution exposed to callers
- [x] Output to `stdout`, logs to `stderr` (STDIO transport requirement)
- [x] Ledger opened read-only (`mode=ro`, `PRAGMA query_only = ON`)

#### Sync preflight (`_vault_preflight`)

- [x] Warns if `VAULT_PATH` not set
- [x] Warns if vault path does not exist
- [x] Warns if tracked files are dirty (`--untracked-files=no` — ignores Obsidian metadata)
- [x] Preflight applied to all vault tools; `get_sync_status` is exempt
- [x] Preflight warns and returns error text — does not auto-sync (sync is `vault-pull`'s job, or `gembrain`'s preflight)

#### Ledger access

- [x] `LEDGER_PATH` env var — optional; `get_sync_status` degrades gracefully if unset
- [x] Read-only SQLite connection (`busy_timeout = 1000`)

## Acceptance criteria

- [x] MCP runs as a host process, not inside Docker — `brain-mcp` is a `pyproject.toml` script entry point invoked directly on the host.
- [x] MCP uses `VAULT_PATH` pointing to the host-visible vault clone.
- [x] `LEDGER_PATH` is optional — `get_sync_status` reports partial data when unset; all other tools are unaffected.
- [x] `read_note` rejects paths outside the vault (traversal), hidden-directory paths, and non-Markdown files.
- [x] Search and list tools enforce result limits — max 100, default 20.
- [x] MCP exposes no mutation tools.
- [x] Preflight checks tracked-file cleanliness but ignores untracked Obsidian metadata (`--untracked-files=no`).
- [ ] Claude Code can read a note filed by the Docker pipeline — requires `LOCAL_VAULT_PATH` bind-mount configured in `compose.override.yaml` so the vault is host-visible. Deployment step, not a code gap; validate during EC2 setup.
