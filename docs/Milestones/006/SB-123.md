# SB-123 — Add the local read-only MCP server

**Branch:** `feature/read-only-mcp`

**Prerequisite:** Implement only after the intake pipeline (SB-101–116), digests (SB-120–121), and vault pull (SB-122) are stable.

## Purpose

Expose the local vault and capture ledger through a narrow read-only MCP server (`brain-mcp`) so local AI clients (Claude Code, Gemini CLI via gembrain) can query notes and tasks without network access to EC2.

---

## Implementation checklist

### MCP server (`src/secondbrain/mcp_server.py`)

- [x] `brain-mcp` entry point in `pyproject.toml`
- [x] STDIO transport via `mcp.server.stdio.stdio_server`
- [x] `mcp>=1.0.0` dependency declared

#### MCP tools

- [x] `search_notes(query, folder?, project?, tags?, limit?)` — keyword search across vault `.md` files; parses frontmatter for project/tags filter; returns `note_path`, `title`, `project`, `tags`
- [x] `read_note(note_path)` — read full content of one note; path is relative to vault root
- [x] `list_recent_notes(days?, folder?, limit?)` — list notes modified within N days; sorted newest-first
- [x] `list_open_tasks(project?, limit?)` — list notes with `status: "open"` actions in frontmatter; delegates to `scan_open_task_list()` in `digest.py`
- [x] `get_sync_status()` — returns vault git head commit, dirty flag, ledger timestamps (`last_successful_reconciliation_at`, `capture_service_state`, `last_successful_backup_at`)

#### Safety constraints (ARCHITECTURE.md §8.14)

- [x] Path-root enforcement (`_enforce_path`) — resolves paths against vault root, rejects traversal
- [x] Hidden-path guard in `read_note` — rejects paths with `.`-prefixed components
- [x] `.md`-only constraint in `read_note`
- [x] Result limit clamped to `[1, 100]` with default 20
- [x] No mutation tools
- [x] No shell execution exposed to callers
- [x] Logs to `stderr`, not `stdout` (STDIO transport requirement)
- [x] Ledger opened read-only (`mode=ro`, `PRAGMA query_only = ON`)

#### Sync preflight (`_vault_preflight`)

- [x] Returns warning string if `VAULT_PATH` not set
- [x] Returns warning string if vault path does not exist
- [x] Returns warning string if git worktree is dirty (run `vault-pull` first)
- [x] Preflight applied to all vault tools before executing; `get_sync_status` is exempt (it reports status, not vault content)
- [ ] **Gap (intentional design):** Preflight warns but does not auto-sync. The full "git fetch + ff-only merge" sequence from ARCHITECTURE.md §8.14 is delegated to `gembrain` (SB-124) which runs vault-pull before invoking any MCP query. The MCP server itself only checks and warns.

#### Ledger access

- [x] `LEDGER_PATH` env var (optional — `get_sync_status` skips ledger if not set)
- [x] Read-only SQLite connection with `busy_timeout = 1000`
- [x] `_ledger_system_state()` helper for `system_state` table reads

### Unit tests (`tests/unit/test_mcp_server.py`)

- [x] Path-root enforcement (normal paths, traversal, double-dot, absolute within vault)
- [x] Result limit clamping (default, over-max, under-one, valid, non-integer)
- [x] `search_notes` — match, no-match, folder filter, project filter, tags filter, limit
- [x] `read_note` — happy path, traversal rejection, hidden path rejection, non-md rejection, missing file
- [x] `list_recent_notes` — recent notes found, older notes excluded, folder filter, mtime sort
- [x] `list_open_tasks` — delegates to `scan_open_task_list`
- [x] `get_sync_status` — vault exists, vault missing, ledger missing
- [x] `_vault_preflight` — vault missing, dirty worktree, clean worktree

### Deployment

- [x] `compose.mcp-local.yaml` — dev-only `mcp-service` profile for Docker exec MCP clients
- [x] Preferred host setup documented: `VAULT_PATH=/path/to/vault brain-mcp` (works with all MCP clients including Claude Code, Claude Desktop)
- [x] MCP client config examples documented (uv run and docker exec variants)
