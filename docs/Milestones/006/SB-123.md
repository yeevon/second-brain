# SB-123 ‚Äî Add the local read-only MCP server

**Release branch:** `milestone_6_2`

**Status:** Implemented and tested.

## Purpose

Expose the local vault and capture ledger through a narrow read-only MCP server (`brain-mcp`) so local AI clients (Claude Code, Gemini CLI via gembrain) can query notes and tasks. Runs as a host process ‚Äî not inside Docker.

---

## Implementation checklist

### MCP server (`src/secondbrain/mcp_server.py`)

- [x] `brain-mcp` entry point in `pyproject.toml`
- [x] STDIO transport (`mcp.server.stdio.stdio_server`)
- [x] `mcp>=1.0.0` dependency

#### Tools

- [x] `search_notes(query, folder?, project?, tags?, limit?)`
- [x] `read_note(note_path)` ‚Äî relative to vault root
- [x] `list_recent_notes(days?, folder?, limit?)`
- [x] `list_open_tasks(project?, limit?)` ‚Äî delegates to `scan_open_task_list()` in `digest.py`
- [x] `get_sync_status()` ‚Äî vault git head, dirty flag, ledger timestamps

#### Safety constraints

- [x] Path-root enforcement ‚Äî resolves paths against vault root; rejects traversal
- [x] Hidden-path guard in `read_note` ‚Äî rejects `.`-prefixed path components
- [x] `.md`-only constraint in `read_note`
- [x] Result limit clamped to `[1, 100]`, default 20
- [x] No mutation tools
- [x] No shell execution exposed to callers
- [x] Output to `stdout`, logs to `stderr` (STDIO transport requirement)
- [x] Ledger opened read-only (`mode=ro`, `PRAGMA query_only = ON`)

#### Sync preflight (`_vault_preflight`)

- [x] Warns if `VAULT_PATH` not set
- [x] Warns if vault path does not exist
- [x] Warns if tracked files are dirty (`--untracked-files=no` ‚Äî ignores Obsidian metadata)
- [x] Preflight applied to all vault tools; `get_sync_status` is exempt
- [x] Preflight warns and returns error text ‚Äî does not auto-sync (sync is `vault-pull`'s job, or `gembrain`'s preflight)

#### Ledger access

- [x] `LEDGER_PATH` env var ‚Äî optional; `get_sync_status` degrades gracefully if unset
- [x] Read-only SQLite connection (`busy_timeout = 1000`)

### Unit tests (`tests/unit/test_mcp_server.py`)

## Acceptance criteria

- [x] MCP runs as a host process, not inside Docker ‚Äî `brain-mcp` is a `pyproject.toml` script entry point invoked directly on the host.
- [x] MCP uses `VAULT_PATH` pointing to the host-visible vault clone.
- [x] `LEDGER_PATH` is optional ‚Äî `get_sync_status` reports partial data when unset; all other tools are unaffected.
- [x] `read_note` rejects paths outside the vault (traversal), hidden-directory paths, and non-Markdown files.
- [x] Search and list tools enforce result limits ‚Äî max 100, default 20.
- [x] MCP exposes no mutation tools.
- [x] Preflight checks tracked-file cleanliness but ignores untracked Obsidian metadata (`--untracked-files=no`).
- [x] Claude Code can read a note filed by the Docker pipeline when `LOCAL_VAULT_PATH` bind-mount mode is configured in `compose.override.yaml`; this remains an operator setup step for each host.
