# SB-120 — Add daily digest workflow

**Release branch:** `milestone_6_2`

## Purpose

Run a daily brief on EC2 that reads current vault state and posts a structured summary to `#brain-digest` or DM. Must work while the local PC is off.

---

## Source of truth

- **Daily focus, due work, upcoming events, birthdays, pending tasks, stale tasks** — vault scan through writer-service (`GET /internal/vault/brief/daily`) proxied by capture-service (`GET /internal/brief/daily`)
- **Legacy capture-count snapshot** — `GET /internal/digest/daily` remains available for count rollups
- **Delivery** — Discord only (webhook URL from n8n environment)

The ledger never exposes raw sensitive content. The vault scan reads frontmatter only.

---

## Config

Configured as n8n environment vars on EC2.

```env
DAILY_DIGEST_ENABLED=true
DISCORD_DIGEST_WEBHOOK_URL=           # read by n8n as $env.DISCORD_DIGEST_WEBHOOK_URL
DIGEST_TIMEZONE=America/Chicago       # used in n8n schedule trigger timezone field
DAILY_DIGEST_TIME=07:00               # configured in n8n schedule trigger node
VAULT_PATH=                           # fallback vault scan path if writer-service is unavailable
```

The schedule trigger timezone and time are set in the n8n workflow JSON, not as runtime env vars. `DISCORD_DIGEST_WEBHOOK_URL` must be available to the n8n container and `N8N_BLOCK_ENV_ACCESS_IN_NODE=false` must allow workflow nodes to read it.
