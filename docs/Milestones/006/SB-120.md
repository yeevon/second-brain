# SB-120 — Add daily digest workflow

**Branch:** `feature/daily-digest`

## Purpose

Run a daily digest on EC2 that reads the capture ledger (and optionally the vault) and posts a structured summary to `#brain-digest` or DM. Must work while the local PC is off.

---

## Source of truth

- **Capture counts, failures, retries, sensitive rejection counts, clarification backlog** — capture ledger (SQLite via capture-service API)
- **Open tasks, filed note paths** — vault scan (`scan_open_tasks` / `scan_open_task_list`)
- **Delivery** — Discord only (webhook URL from n8n variable)

The ledger never exposes raw sensitive content. The vault scan reads frontmatter only.

---

## Config

Configured as n8n workflow variables and environment vars on EC2.

```env
DAILY_DIGEST_ENABLED=true
DISCORD_DIGEST_WEBHOOK_URL=           # n8n workflow variable ($vars.DISCORD_DIGEST_WEBHOOK_URL)
DIGEST_TIMEZONE=America/Chicago       # used in n8n schedule trigger timezone field
DAILY_DIGEST_TIME=07:00               # configured in n8n schedule trigger node
VAULT_PATH=                           # required for open_tasks_count; digest degrades gracefully without it
```

The schedule trigger timezone and time are set in the n8n workflow JSON, not as runtime env vars. `DISCORD_DIGEST_WEBHOOK_URL` is an n8n-level variable, not a capture-service env var.

## Acceptance criteria

- [x] Given captures exist in the last daily window, the digest includes counts and paths for filed notes.
- [x] Given failed/retry captures exist, the digest lists them without raw stack traces — counts only.
- [x] Given sensitive rejections exist, the digest reports counts only, never plaintext.
- [ ] Given open tasks exist in vault frontmatter, the digest includes them grouped by project/folder — currently reports total count only.
- [ ] If no activity exists, send a short "No new activity" digest or skip — not yet implemented.
- [ ] Digest delivery failure is logged and retried or visibly reported — n8n Error Trigger not yet wired to digest workflow.
