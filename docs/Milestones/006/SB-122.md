# SB-122 — Add local pull-only Obsidian sync wrapper

**Branch:** `feature/local-vault-pull`

## Purpose

Provide a host-side `vault-pull` CLI command that syncs the local Obsidian clone from GitHub in pull-only mode. Must fail visibly on a dirty worktree or a diverged history so the user never silently ends up with stale or conflicted vault content.

Version one is pull-only. Two-way sync is explicitly deferred until a conflict strategy is designed (ARCHITECTURE.md §8.9).

---

## Git sequence (from ARCHITECTURE.md §8.9)

```
git fetch origin
git merge --ff-only origin/main
verify clean worktree
fail visibly on dirty tree or conflict
```

The worktree check runs **before** the fetch so a dirty local state is caught immediately without making a network call first.

- [x] Local sync fails visibly on dirty worktree — user sees the offending file list

---

## Design invariants (ARCHITECTURE.md §14)

- **Pull-only.** `vault-pull` never pushes, never commits, never modifies the remote.
- **No blind overwrites.** A dirty worktree is rejected before fetching — the user must resolve local state first.
- **No auto-resolution.** A non-fast-forward merge is an error, not something to rebase or force through.
