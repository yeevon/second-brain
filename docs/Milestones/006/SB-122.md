# SB-122 ‚Äî Local pull-only Obsidian sync wrapper

**Release branch:** `milestone_6_2`

**Status:** Implemented and tested.

## Purpose

Provide a host-side `vault-pull` CLI command that syncs the local Obsidian clone from GitHub in pull-only mode. Fails visibly on dirty worktree or diverged history. Version one is pull-only ‚Äî two-way sync deferred until a conflict strategy is designed.

---

## Git sequence

```
check worktree for dirty tracked files ‚Üí fail visibly if dirty
git fetch origin
git merge --ff-only origin/main       ‚Üí fail on diverged history or conflict
```

The worktree check runs before the fetch so dirty state is caught without a network call.

---

## Exit codes

| Code | Meaning                                                    |
|------|------------------------------------------------------------|
| `0`  | Pull succeeded                                             |
| `1`  | Vault path does not exist, or `VAULT_PATH` env var not set |
| `2`  | Dirty worktree ‚Äî tracked files have uncommitted changes    |
| N    | Git exit code propagated from failed fetch or merge        |

## Acceptance criteria

- [x] `vault-pull` fails before fetch/merge if tracked files are dirty.
- [x] `vault-pull` fetches origin and fast-forward merges `origin/main`.
- [x] `vault-pull` refuses non-fast-forward or conflict states.
- [x] Untracked Obsidian metadata does not block pull ‚Äî the dirty check uses `--untracked-files=no`, consistent with the MCP preflight.
- [x] Exit codes are documented (see table above).

---

## Design invariants

- **Pull-only.** `vault-pull` never pushes, commits, or modifies the remote.
- **No blind overwrites.** Dirty worktree is rejected before fetching.
- **No auto-resolution.** Non-fast-forward is an error, not a rebase opportunity.
