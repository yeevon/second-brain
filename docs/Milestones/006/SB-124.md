# SB-124 — Add gembrain CLI wrapper and Gemini CLI integration

**Branch:** `feature/gembrain-cli`

**Prerequisite:** SB-122 (vault-pull) and SB-123 (brain-mcp) must be complete.

## Purpose

Provide a single `gembrain` host command that:

1. Runs a vault-pull preflight (git fetch + ff-only merge + clean-worktree check) before any query.
2. Wraps the `brain-mcp` MCP tools as direct sub-commands for common queries.
3. Integrates with the Gemini CLI so `gembrain ask "..."` invokes Gemini with `brain-mcp` as its MCP tool source.

Reference: ARCHITECTURE.md §8.14 ("Local MCP server and Gemini CLI wrapper").

---

## Commands to implement

```
gembrain status
gembrain recent [--days N] [--folder F] [--limit N]
gembrain tasks [--project P] [--limit N]
gembrain ask "<natural language question>"
```

---

## Implementation checklist

### Entry point

- [ ] `src/secondbrain/gembrain.py` — main CLI module
- [ ] `gembrain = "secondbrain.gembrain:main"` added to `[project.scripts]` in `pyproject.toml`

### Vault-pull preflight (run before every command except `status`)

The preflight must match the sequence from ARCHITECTURE.md §8.14:

- [ ] `git fetch origin` — refresh remote refs
- [ ] `git merge --ff-only origin/main` — apply remote changes; fail loudly on diverged history
- [ ] Verify clean worktree — fail if dirty after merge (unexpected local edits)
- [ ] On any preflight failure: print a clear error and exit non-zero; do not proceed with the query
- [ ] Reuse `pull_vault()` from `vault_pull.py` (do not duplicate the git logic)

`gembrain status` is exempt from the preflight — it must report vault state even when the vault is stale or unreachable.

### `gembrain status`

Report the following fields (ARCHITECTURE.md §8.14):

- [ ] Local HEAD commit (short hash + subject)
- [ ] Remote HEAD commit (short hash + subject via `git rev-parse origin/main`)
- [ ] Whether local == remote (up to date / N commits behind)
- [ ] Dirty worktree flag
- [ ] Last successful pull timestamp (from `system_state` via `vault-pull` writing to it, or from ledger)
- [ ] Last successful MCP query timestamp (from a local state file or env)
- [ ] Pending conflict flag (detects `.git/MERGE_HEAD`)

Exit code: `0` if vault is clean and in sync, `1` otherwise.

### `gembrain recent [--days N] [--folder F] [--limit N]`

- [ ] Run vault-pull preflight
- [ ] Call `_do_list_recent_notes(vault_path, days=N, folder=F, limit=L)` directly (no MCP subprocess needed — import from `mcp_server`)
- [ ] Print results as a human-readable table or list to stdout
- [ ] Default: `--days 7`, `--limit 20`

### `gembrain tasks [--project P] [--limit N]`

- [ ] Run vault-pull preflight
- [ ] Call `scan_open_task_list(vault_path, project=P, limit=L)` directly (import from `digest`)
- [ ] Print results grouped by note, with action text, to stdout
- [ ] Default: `--limit 20`

### `gembrain ask "<question>"`

- [ ] Run vault-pull preflight
- [ ] Invoke the `gemini` CLI with `brain-mcp` configured as the MCP server:
  ```
  gemini --mcp-server "brain-mcp" "<question>"
  ```
  Or via the MCP config approach (whichever the Gemini CLI supports)
- [ ] `VAULT_PATH` and `LEDGER_PATH` must be forwarded to the `brain-mcp` subprocess environment
- [ ] If `gemini` CLI is not found on `PATH`, exit with a clear error message pointing to installation instructions
- [ ] Pass the question as the prompt; let Gemini decide which MCP tools to call

### Error handling

- [ ] All errors go to stderr
- [ ] All structured output goes to stdout
- [ ] Non-zero exit code on: preflight failure, missing `VAULT_PATH`, `gemini` not found, MCP tool error
- [ ] Clear human-readable error messages — no Python tracebacks shown to the user

### Environment variables

- [ ] `VAULT_PATH` — required for all commands (except `status` which degrades gracefully)
- [ ] `LEDGER_PATH` — optional; used by `status` to read ledger timestamps

### Unit tests (`tests/unit/test_gembrain.py`)

- [ ] `gembrain status` — vault clean and synced, dirty vault, no VAULT_PATH
- [ ] `gembrain status` — local == remote, local behind remote
- [ ] `gembrain recent` — runs preflight, calls list_recent_notes, formats output
- [ ] `gembrain recent` — preflight failure aborts before query
- [ ] `gembrain tasks` — runs preflight, calls scan_open_task_list, formats output
- [ ] `gembrain tasks` — project filter passed through
- [ ] `gembrain ask` — runs preflight, constructs correct gemini command
- [ ] `gembrain ask` — gemini not found → clear error, non-zero exit
- [ ] `gembrain ask` — preflight failure aborts before gemini invocation
- [ ] VAULT_PATH not set → `status` degrades, all other commands exit non-zero with message

## What is NOT in this ticket

- Vector search / embeddings → explicitly deferred (ARCHITECTURE.md §16)
- Writable MCP tools → explicitly deferred
- `gembrain` writing anything to the vault → read-only constraint enforced by `brain-mcp`
- Custom Gemini API calls from within gembrain (use the Gemini CLI, not the API directly)
