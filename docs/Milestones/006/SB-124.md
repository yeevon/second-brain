# SB-124 — Add gembrain CLI wrapper and Gemini CLI integration

**Release branch:** `milestone_6_2`

**Prerequisite:** SB-122 (vault-pull) and SB-123 (brain-mcp) must be complete.

**Status:** Implemented and tested.

## Purpose

Provide a single `gembrain` host command that:

1. Runs a vault-pull preflight (git fetch + ff-only merge + clean-worktree check) before any query.
2. Wraps the `brain-mcp` MCP tools as direct sub-commands for common queries.
3. Integrates with the Gemini CLI so `gembrain ask "..."` invokes Gemini with `brain-mcp` as its MCP tool source.

Reference: ARCHITECTURE.md §8.14 ("Local MCP server and Gemini CLI wrapper").

---

## Commands to implement

```
gembrain status
gembrain recent [--days N] [--folder F] [--limit N]
gembrain tasks [--project P] [--limit N]
gembrain ask "<natural language question>"
```

## Implementation checklist

### Entry point

- [x] `src/secondbrain/gembrain.py` — main CLI module
- [x] `gembrain = "secondbrain.gembrain:main"` added to `[project.scripts]` in `pyproject.toml`

### Vault-pull preflight (run before every command except `status`)

The preflight must match the sequence from ARCHITECTURE.md §8.14:

- [x] `git fetch origin` — refresh remote refs
- [x] `git merge --ff-only origin/main` — apply remote changes; fail loudly on diverged history
- [x] Verify clean worktree — fail if dirty after merge (unexpected local edits)
- [x] On any preflight failure: print a clear error and exit non-zero; do not proceed with the query
- [x] Reuse `pull_vault()` from `vault_pull.py` (do not duplicate the git logic)

`gembrain status` is exempt from the preflight — it must report vault state even when the vault is stale or unreachable.

### `gembrain status`

Report the following fields (ARCHITECTURE.md §8.14):

- [x] Local HEAD commit (short hash + subject)
- [x] Remote HEAD commit (short hash + subject from `origin/main`)
- [x] Whether local == remote (up to date / N commits behind)
- [x] Dirty worktree flag
- [x] Last successful pull timestamp when available from the ledger
- [x] Last successful MCP query timestamp when available from the ledger
- [x] Pending conflict flag (detects `.git/MERGE_HEAD`)

Exit code: `0` if vault is clean and in sync, `1` otherwise.

### `gembrain recent [--days N] [--folder F] [--limit N]`

- [x] Run vault-pull preflight
- [x] Call `_do_list_recent_notes(vault_path, days=N, folder=F, limit=L)` directly (no MCP subprocess needed — import from `mcp_server`)
- [x] Print results as a human-readable table or list to stdout
- [x] Default: `--days 7`, `--limit 20`

### `gembrain tasks [--project P] [--limit N]`

- [x] Run vault-pull preflight
- [x] Call `scan_open_task_list(vault_path, project=P, limit=L)` directly (import from `digest`)
- [x] Print results grouped by note, with action text, to stdout
- [x] Default: `--limit 20`

### `gembrain ask "<question>"`

- [x] Run vault-pull preflight
- [x] Invoke the `gemini` CLI with `brain-mcp` configured as the MCP server:
  ```
  gemini --mcp-server "brain-mcp" "<question>"
  ```
  Or via the MCP config approach (whichever the Gemini CLI supports)
- [x] `VAULT_PATH` and `LEDGER_PATH` must be forwarded to the `brain-mcp` subprocess environment
- [x] If `gemini` CLI is not found on `PATH`, exit with a clear error message pointing to installation instructions
- [x] Pass the question as the prompt; let Gemini decide which MCP tools to call

### Error handling

- [x] All errors go to stderr
- [x] All structured output goes to stdout
- [x] Non-zero exit code on: preflight failure, missing `VAULT_PATH`, `gemini` not found, MCP tool error
- [x] Clear human-readable error messages — no Python tracebacks shown to the user

### Environment variables

- [x] `VAULT_PATH` — required for all commands (except `status` which degrades gracefully)
- [x] `LEDGER_PATH` — optional; used by `status` to read ledger timestamps

## What is NOT in this ticket

- Writable MCP tools → explicitly deferred
- `gembrain` writing anything to the vault → read-only constraint enforced by `brain-mcp`
- Custom Gemini API calls from within gembrain (use the Gemini CLI, not the API directly)
