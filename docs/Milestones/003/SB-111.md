# SB-111 — Deploy a secured n8n foundation

**Branch:** `feature/n8n-foundation`

Add a single secured n8n instance alongside `capture-service`. Do not build the capture-forwarding workflow yet. This story establishes the durable orchestration platform, private UI access, security baseline, persistence guarantees, and Error Trigger workflow required by later stories.

---

## Goal

After SB-111:

```text
EC2
├── capture-service
│   ├── owns the durable capture SQLite ledger
│   ├── listens to Discord
│   └── exposes internal API on backend network only
│
└── n8n
    ├── stores workflows and credentials persistently
    ├── reaches capture-service over the private Compose network
    ├── exposes its editor UI only on EC2 loopback
    └── is accessed from the desktop through an SSH tunnel
```

The operator must be able to:

```text
restart the Compose stack
    ↓
log back into n8n
    ↓
see the same workflows
    ↓
see the same credentials
    ↓
retain the same encryption key
```

---

## Scope decision: private SSH-tunnel access

Use an SSH tunnel for SB-111. Do not add:

```text
public port 5678
public port 443
public DNS
Caddy
Traefik
Cloudflare Tunnel
Tailscale Serve
public webhook endpoints
```

The existing EC2 security-group rule remains:

```text
SSH 22 from the operator's intentional /32 source
```

n8n binds to host loopback only:

```text
127.0.0.1:5678
```

The desktop reaches it through:

```bash
ssh -N \
  -L 5678:127.0.0.1:5678 \
  ubuntu@<EC2_HOST>
```

The editor is then opened locally at:

```text
http://127.0.0.1:5678
```

This is intentionally a private development-access layer. The browser reaches `http://localhost:5678` through an encrypted SSH tunnel, so HTTPS-only cookies would not work. `N8N_SECURE_COOKIE=false` is deliberate and documented. If HTTPS is added later, flip it to `true` and add the reverse-proxy settings: `WEBHOOK_URL`, `N8N_PROXY_HOPS=1`, and forwarded headers.

## 1. Add the n8n Compose overlay

Create `compose.n8n.yaml`. Use this shape:

```yaml
services:
  n8n:
    image: docker.n8n.io/n8nio/n8n:${N8N_IMAGE_TAG:?N8N_IMAGE_TAG must be set}
    container_name: second-brain-n8n
    restart: unless-stopped
    init: true
    env_file:
      - ${N8N_ENV_FILE:?N8N_ENV_FILE must be set}
    environment:
      N8N_ENCRYPTION_KEY_FILE: /run/secrets/n8n_encryption_key
    secrets:
      - n8n_encryption_key
    volumes:
      - ${N8N_DATA_SOURCE:?N8N_DATA_SOURCE must be set}:/home/node/.n8n
    ports:
      - "127.0.0.1:5678:5678"
    expose:
      - "5678"
    networks:
      - backend
    cap_drop:
      - ALL
    security_opt:
      - no-new-privileges:true
    healthcheck:
      test:
        [
          "CMD",
          "node",
          "-e",
          "fetch('http://127.0.0.1:5678/').then(r => process.exit(r.status < 500 ? 0 : 1)).catch(() => process.exit(1))",
        ]
      interval: 30s
      timeout: 5s
      retries: 5
      start_period: 30s

secrets:
  n8n_encryption_key:
    file: ${N8N_ENCRYPTION_KEY_FILE:?N8N_ENCRYPTION_KEY_FILE must be set}
```

### Requirements

- Do not use an untagged image.
- Do not use `latest`.
- Pin a tested stable n8n release.
- Do not publish `0.0.0.0:5678`.
- Do not add public inbound security-group rules for `5678`.
- Do not attach n8n to a public-facing Compose network.
- Do not give n8n direct access to the capture-service SQLite file.
- Do not mount the Obsidian vault into n8n.

The only n8n-to-capture-service dependency allowed in this story is HTTP over:

```text
http://capture-service:8000
```

## 4. Store the n8n encryption key outside the repository

Create the host secret file:

```text
/opt/second-brain/config/n8n-encryption-key
```

Generate it once:

```bash
openssl rand -hex 32
```

Install it with restrictive permissions:

```bash
install \
  -m 600 \
  /dev/null \
  /opt/second-brain/config/n8n-encryption-key
nano \
  /opt/second-brain/config/n8n-encryption-key
```

Populate only the generated value. Do not:

```text
commit the key
print the key
log the key
rotate the key casually
store the key in README examples
store the key in the Docker image
store the key in workflow JSON
```

Use the Compose secret mount and `N8N_ENCRYPTION_KEY_FILE=/run/secrets/n8n_encryption_key`.

Add the local secret file patterns to `.gitignore` and `.dockerignore`:

```text

n8n-encryption-key.local
```

## 5. Add the n8n environment template

Create `deploy/n8n.env.example`:

```dotenv
# Image is pinned outside this file through N8N_IMAGE_TAG.

# Runtime
TZ=America/Chicago
GENERIC_TIMEZONE=America/Chicago
N8N_HOST=localhost
N8N_PORT=5678
N8N_LISTEN_ADDRESS=0.0.0.0
N8N_PROTOCOL=http

# Private SSH-tunnel mode only.
#
# The editor is accessed through:
#   ssh -N -L 5678:127.0.0.1:5678 ubuntu@<EC2_HOST>
#
# The browser sees http://127.0.0.1:5678, so HTTPS-only cookies cannot be
# used in this mode. Do not publish port 5678 publicly. If HTTPS is added
# later, set N8N_SECURE_COOKIE=true and configure reverse-proxy settings.
N8N_SECURE_COOKIE=false
N8N_SAMESITE_COOKIE=lax

# Credentials and local configuration hardening
N8N_ENFORCE_SETTINGS_FILE_PERMISSIONS=true
N8N_BLOCK_ENV_ACCESS_IN_NODE=true
N8N_BLOCK_FILE_ACCESS_TO_N8N_FILES=true

# Keep workflows simple while the pipeline is under construction
N8N_CONCURRENCY_PRODUCTION_LIMIT=1

# Recommended n8n task execution path
N8N_RUNNERS_ENABLED=true

# Limit retained execution payloads.
# Both defaults are deliberately none. SB-112 adds workflows that temporarily
# handle raw capture text. Retaining error or manual executions at the global
# level would expose that text in n8n storage. Workflow-level overrides in
# SB-112 provide defense-in-depth, but the global defaults are the primary
# control. Do not weaken these for debugging — use synthetic captures instead.
EXECUTIONS_DATA_SAVE_ON_ERROR=none
EXECUTIONS_DATA_SAVE_ON_SUCCESS=none
EXECUTIONS_DATA_SAVE_ON_PROGRESS=false
EXECUTIONS_DATA_SAVE_MANUAL_EXECUTIONS=false
EXECUTIONS_DATA_PRUNE=true
EXECUTIONS_DATA_MAX_AGE=168
EXECUTIONS_DATA_PRUNE_MAX_COUNT=1000

# Reduce unnecessary external behavior
N8N_DIAGNOSTICS_ENABLED=false
N8N_PERSONALIZATION_ENABLED=false
N8N_TEMPLATES_ENABLED=false

# Disable unused public API surface during the foundation phase
N8N_PUBLIC_API_DISABLED=true
N8N_PUBLIC_API_SWAGGERUI_DISABLED=true

# Keep security-update notices visible in the UI
N8N_VERSION_NOTIFICATIONS_ENABLED=true

# Explicit shutdown allowance
N8N_GRACEFUL_SHUTDOWN_TIMEOUT=30
```

Do not add these in SSH-tunnel mode:

```text
WEBHOOK_URL
N8N_EDITOR_BASE_URL
N8N_PROXY_HOPS
```

They become relevant only when a reverse proxy or HTTPS access layer exists.

## 8. Add an SSH-tunnel helper

Create `deploy/open-n8n-tunnel.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

HOST="${1:?Usage: deploy/open-n8n-tunnel.sh <EC2_HOST> [SSH_USER]}"
SSH_USER="${2:-ubuntu}"

exec ssh \
  -N \
  -L 5678:127.0.0.1:5678 \
  "$SSH_USER@$HOST"
```

Run this script from the desktop, not from EC2. Open `http://127.0.0.1:5678` after the tunnel connects.

## 10. Add a one-time idempotent bootstrap script

Create `deploy/bootstrap-n8n.sh`. The script must:

```text
require the n8n container to be healthy
query existing workflows by exact name before importing
skip import when "Second Brain - Error Handler" already exists
when absent, import using the n8n container CLI:
    query existing workflow names to confirm absence before importing:
        docker exec second-brain-n8n \
          n8n export:workflow --all --output=/tmp/existing-workflows.json
        docker cp \
          second-brain-n8n:/tmp/existing-workflows.json \
          "$TMP_DIR/existing-workflows.json"
        jq -r '.[].name' "$TMP_DIR/existing-workflows.json"
        print only names or counts — never the full JSON
    sanitize the fixture by stripping id and versionId before import:
        jq 'del(.id, .versionId)' \
          n8n/workflows/second-brain-error-handler.json \
          > "$TMP_DIR/bootstrap-error-handler.json"
    copy the sanitized fixture into the container:
        docker cp "$TMP_DIR/bootstrap-error-handler.json" \
          second-brain-n8n:/tmp/bootstrap-error-handler.json
    import using the n8n CLI inside the container:
        docker exec second-brain-n8n \
          n8n import:workflow --input=/tmp/bootstrap-error-handler.json
    remove the temporary files:
        docker exec --user root second-brain-n8n \
          rm -f /tmp/bootstrap-error-handler.json \
                /tmp/existing-workflows.json
    query workflows by exact name to confirm the import succeeded
    exit non-zero if the workflow name is not found after import
exit non-zero if a duplicate workflow name is detected at any point
leave all imported workflows inactive — do not activate them
print whether each workflow was imported or skipped
never overwrite operator-created credentials
```

Do not run bootstrap automatically on every container restart. The operator runs it deliberately after creating the initial owner account. Imported workflows remain inactive until the operator binds the required credentials manually in the UI.
