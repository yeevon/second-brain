# SB-119 — Add encrypted off-host backups and restore validation

**Branch:** `feature/encrypted-backups`

---

## What to back up

- SQLite ledger — using SQLite-safe backup tooling (not a raw copy of a live database file).
- EC2 vault clone.
- n8n data volume.
- Service configuration without plaintext secrets where practical.

## Schedule

- Nightly encrypted snapshot.
- Weekly restore test or validation.

The architecture treats GitHub as a remote replica, not the only backup.

---

## Acceptance criteria

- SQLite backup uses `.backup` or equivalent safe tooling — never a raw `cp` of a live `.db` file.
- Restore validation runs into a temporary directory or container only. It never touches live volumes.
- Backup output is encrypted before leaving the host.
- Secrets are excluded or redacted in service configuration backups where practical.
- `sb status` reports last successful backup timestamp and last successful restore validation timestamp.
- A restore script can validate a backup into a temporary location without affecting the live deployment.

**Done when:** all acceptance criteria above pass and backup/restore timestamps are visible in status output.
