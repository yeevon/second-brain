# SB-139 — Configuration preflight command

**Branch:** `feature/config-preflight`
**Source:** TD-P1-004

---

## Goal

Operators need a safe way to detect missing or invalid configuration before deployment. Currently misconfiguration is only discovered at runtime, often in confusing ways.

---

## Acceptance criteria

- [ ] Add `uv run secondbrain config-check` command.
- [ ] Reports missing required variables by name only — no values printed.
- [ ] Reports renamed variables where applicable (e.g., old name → new name).
- [ ] Reports which `.env` template covers each variable.
- [ ] Distinguishes `local-full` vs `capture-only` requirements.
- [ ] Validates token minimum lengths (no value printed).
- [ ] Validates numeric ranges.
- [ ] Never prints secret values.
- [ ] Exits non-zero on any failure.
- [ ] Does not replace the runtime fail-closed validation — both must coexist.
- [ ] Tests cover:
  - All required vars present — exit 0
  - Missing required var — exit non-zero, name printed, value not printed
  - Token too short — exit non-zero, name printed, value not printed
  - Numeric out of range — exit non-zero

**Done when:** `config-check` can be run safely before `docker compose up` and catches all misconfiguration without leaking secret values.
