# SB-137 — Background task liveness monitoring

**Branch:** `feature/background-task-liveness`
**Source:** TD-P1-002

---

## Goal

Background tasks can die silently. `uv run secondbrain status` has no way to tell an operator whether critical background tasks are alive or how recently they last ran successfully.

Tracked tasks:
- Capture-service heartbeat
- Periodic Discord reconciliation
- Stale-processing-lease reaper
- Local classifier worker
- Downstream delivery dispatcher

---

## Acceptance criteria

- [ ] Each task records a last-successful-pass timestamp on every healthy iteration.
- [ ] A task that has not passed within a configurable threshold is marked `degraded`.
- [ ] `uv run secondbrain status` reports each task:
  - Task name
  - Status: `running` / `completed_unexpectedly` / `degraded`
  - Last successful pass timestamp
  - Last safe error type (no raw exception text)
- [ ] Status output contains numeric metadata and safe error categories only — no raw exception bodies.
- [ ] Tests cover:
  - Healthy task shows `running` with recent timestamp
  - Task that exited unexpectedly shows `completed_unexpectedly`
  - Task that has not passed recently shows `degraded`

**Done when:** an operator can run `status` and immediately see which background tasks are healthy and when they last succeeded.
