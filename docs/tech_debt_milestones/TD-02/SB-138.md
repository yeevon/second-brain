# SB-138 — Shutdown cleanup resilience

**Branch:** `feature/shutdown-cleanup-resilience`
**Source:** TD-P1-003

---

## Goal

One cleanup failure during shutdown can prevent later cleanup steps from running, potentially leaving resources unreleased. Each step must be isolated so that a failure in one does not block the rest.

---

## Acceptance criteria

- [ ] Each shutdown step runs in its own `try/except`.
- [ ] Failure logs include the cleanup step name and error type only — no raw exception text.
- [ ] SQLite close always runs, even if earlier steps fail.
- [ ] Cleanup failures are logged, not swallowed.
- [ ] Existing shutdown tests continue to pass.
- [ ] Tests cover:
  - First cleanup step fails — remaining steps still run
  - SQLite close runs even after a failed step

**Done when:** a shutdown with one failed step still completes all remaining cleanup.
