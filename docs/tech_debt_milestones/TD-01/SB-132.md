# SB-132 — Gate correction commands through capture authorization

**Branch:** `feature/correction-auth-gate`
**Source:** TD-P0-003

---

## Goal

Correction commands (`fix:`, `fix SB-...:`) currently may bypass the guild/channel/user/bot authorization rules applied to normal captures. An unauthorized user in an unauthorized channel must not be able to trigger a correction.

---

## Acceptance criteria

- [ ] `fix:` and `fix SB-...:` commands are ignored unless the message passes normal capture authorization.
- [ ] Unauthorized guild/channel/user correction attempts produce no state change and no receipt.
- [ ] Tests cover:
  - Unauthorized guild — no correction applied
  - Unauthorized channel — no correction applied
  - Unauthorized user — no correction applied
  - Bot/self message — no correction applied
  - Authorized sender — correction proceeds normally
- [ ] Authorized correction behavior is unchanged.

**Done when:** correction commands share the exact same authorization path as normal capture intake.
