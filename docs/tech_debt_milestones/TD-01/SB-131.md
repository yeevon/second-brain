# SB-131 — Sanitize persisted local-worker exception messages

**Branch:** `feature/sanitize-local-worker-errors`
**Source:** TD-P0-002

---

## Goal

The local classifier worker can persist raw exception text into SQLite. Exception strings may contain tokens, internal URLs, raw HTTP bodies, note content, or unbounded stack/error text — all of which violate the invariant that persisted fields contain only sanitized metadata.

---

## Acceptance criteria

- [x] Add a shared sanitizer for local-worker failure paths.
- [x] Persisted failure fields contain only:
  - Safe error category
  - Exception class name
  - Optional bounded summary (no raw content)
- [x] Persisted failure fields never contain:
  - Tokens or credentials
  - Internal URLs
  - Raw HTTP response bodies
  - Raw note text
  - Unbounded exception strings
- [x] Remove all raw `str(exc)` from persisted failure fields.
- [x] Unit tests cover:
  - Secret-looking strings in exception message
  - URLs in exception message
  - Long exception text (truncation behavior)
  - Vault failures
  - Unexpected worker failures

**Done when:** no local-worker failure path can persist a token, URL, or raw note text into SQLite.
