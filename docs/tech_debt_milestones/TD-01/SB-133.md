# SB-133 — Resolve clarifications only after correction succeeds

**Branch:** `feature/correction-clarification-ordering`
**Source:** TD-P0-004

---

## Goal

Clarifications must not be marked resolved until the correction has actually succeeded. The current ordering risks marking a clarification resolved even when the downstream correction fails, leaving the capture in an incorrect state with no visible indication.

---

## Acceptance criteria

- [ ] `resolve_clarification()` runs only after `apply_correction()` returns a successful result.
- [ ] A failed correction leaves clarification status unchanged.
- [ ] Tests cover:
  - Writer-service correction failure — clarification remains unresolved
  - Successful correction — clarification is resolved
  - Network failure during correction — clarification remains unresolved

**Done when:** no code path can resolve a clarification before correction success is confirmed.
