# SB-130 — WAL-safe SQLite backup and restore procedure

**Branch:** `feature/wal-safe-backup`
**Source:** TD-P0-001

---

## Goal

SQLite WAL mode means copying only `ledger.sqlite3` may miss committed data still present in `ledger.sqlite3-wal`. Define and test a trustworthy backup and restore path before any production data loss event occurs.

---

## Acceptance criteria

- [x] Define a WAL-safe backup procedure using at least one of:
  - SQLite online backup API via `sqlite3.Connection.backup()`
  - Controlled checkpoint followed by filesystem snapshot
  - Quiesced EBS snapshot
- [x] Add a restore test that:
  - Creates a backup
  - Restores to a separate path
  - Verifies row counts and integrity checks pass
- [x] Document the procedure in `deploy/README.md`.
- [x] Do not add automatic backup infrastructure until the manual procedure has passed restore testing.

**Done when:** a backup taken during live operation can be restored to a separate path and all row counts and integrity checks pass.
