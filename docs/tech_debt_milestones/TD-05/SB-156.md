# SB-156 — Defensive SQLiteRuntime constructor validation

**Branch:** `fix/sqlite-runtime-constructor-validation`
**Source:** TD-P3-008

---

## Goal

The `SQLiteRuntime` constructor does not validate its arguments defensively. Invalid inputs may produce confusing errors at runtime rather than at construction time.

> **Evaluate after SB-136 (SQLite watchdogs).** If watchdog work adds constructor validation naturally, fold this into that ticket rather than implementing separately.

---

## Acceptance criteria

- [ ] `SQLiteRuntime` constructor validates required arguments.
- [ ] Invalid inputs raise a clear `ValueError` at construction time.
- [ ] If this is fully covered by SB-136, close as duplicate and document the resolution.

**Done when:** invalid `SQLiteRuntime` construction fails immediately with a clear message, or the work is confirmed covered by SB-136.
