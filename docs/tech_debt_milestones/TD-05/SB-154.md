# SB-154 — Rename duplicate delivery acceptance log noise

**Branch:** `fix/duplicate-delivery-log`
**Source:** TD-P3-006 / GitHub #8

---

## Goal

`duplicate_delivery_acceptance_ignored` is logged for normal idempotent replay behavior, but the name looks like an error. Operators investigating logs waste time on a non-issue.

---

## Acceptance criteria

- [ ] Rename or downgrade the event to a name that signals expected behavior.
- [ ] Event includes `outcome=idempotent_replay`.
- [ ] Document when this event is expected (in a comment near the log call or in `deploy/README.md`).

**Done when:** the event name does not suggest a problem and the outcome field makes idempotent replay explicit.
