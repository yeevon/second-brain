# SB-154 — Rename duplicate delivery acceptance log noise

**Branch:** `fix/duplicate-delivery-log`
**Source:** TD-P3-006 / GitHub #8

---

## Goal

`duplicate_delivery_acceptance_ignored` is logged for normal idempotent replay behavior, but the name looks like an error. Operators investigating logs waste time on a non-issue.

---

## Implementation

**File:** `src/secondbrain/delivery.py`

Rename the event at the one call site (line ~155):

```python
# Before:
log_metadata(
    "duplicate_delivery_acceptance_ignored",
    capture_id=capture.capture_id,
    delivery_attempt=attempt,
)

# After:
log_metadata(
    "delivery_idempotent_replay_accepted",
    capture_id=capture.capture_id,
    delivery_attempt=attempt,
    outcome="idempotent_replay",
)
```

**Required fields on the renamed event:**

| Field | Value |
|---|---|
| `capture_id` | the capture ID string |
| `delivery_attempt` | the integer delivery attempt number |
| `outcome` | literal string `"idempotent_replay"` |

No other call sites use `duplicate_delivery_acceptance_ignored` (confirmed: only one occurrence in `src/`).
