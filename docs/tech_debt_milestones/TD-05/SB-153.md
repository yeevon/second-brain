# SB-153 — Reduce local startup reconciliation surprises

**Branch:** `fix/local-startup-reconciliation`
**Source:** TD-P3-005 / GitHub #8

---

## Goal

Manual local validation is confusing when startup reconciliation captures existing Discord messages in the test channel. Developers expect a clean run but get captures from previous sessions.

---

## Decision

**Selected path: add `STARTUP_RECONCILE_ENABLED` env flag to disable startup reconciliation for smoke tests.**

Rationale: documentation-only is easy to miss. A log warning helps but does not prevent surprise captures from clogging the ledger. The env flag gives smoke-test operators a deterministic clean-slate experience without changing production behavior.

---

## Contract

### Env var

`STARTUP_RECONCILE_ENABLED` — values: `true` | `false`

- Default: `true` (no behavior change when unset)
- `false` — skips the `startup_reconcile` call entirely on service start; logs `startup_reconcile_disabled_for_local_smoke` once at startup

### Where the check lives

**File:** `src/secondbrain/app.py` (or wherever `startup_reconcile` is called during service init)

```python
if settings.startup_reconcile_enabled:
    await service.startup_reconcile(client)
else:
    log_metadata("startup_reconcile_disabled_for_local_smoke")
```

### Settings parsing

**File:** `src/secondbrain/config.py`

```python
self.startup_reconcile_enabled = _parse_bool_env("STARTUP_RECONCILE_ENABLED", default=True)
```

Parsing rules (strict):

- Unset or empty string → `True` (default)
- `"true"` (case-insensitive) → `True`
- `"false"` (case-insensitive) → `False`
- Any other value → `ValueError("STARTUP_RECONCILE_ENABLED must be 'true' or 'false', got ...")`

Use whatever `_parse_bool_env` helper already exists, or add one that follows these exact rules. Do not silently coerce unrecognized values to `True`.

### Production default

`STARTUP_RECONCILE_ENABLED` is not set in `compose.yaml` or `compose.n8n.yaml`. Default `true` keeps production behavior unchanged.

Local smoke profile (`compose.override.yaml`) does not set it either — operators opt in explicitly via their `.env` file or shell export. This is intentional: the flag is for individual developers, not a compose default.

---

## Acceptance criteria

- [ ] `STARTUP_RECONCILE_ENABLED=false` skips `startup_reconcile` and logs `startup_reconcile_disabled_for_local_smoke`.
- [ ] `STARTUP_RECONCILE_ENABLED=true` (or unset) behaves identically to current behavior.
- [ ] Config validation rejects values other than `true`/`false` (case-insensitive) with a clear `ValueError` naming the variable.
- [ ] Unset or empty defaults to `true` without error.
