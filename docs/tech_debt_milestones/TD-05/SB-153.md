# SB-153 — Reduce local startup reconciliation surprises

**Branch:** `fix/local-startup-reconciliation`
**Source:** TD-P3-005 / GitHub #8

---

## Goal

Manual local validation is confusing when startup reconciliation captures existing Discord messages in the test channel. Developers expect a clean run but get captures from previous sessions.

---

## Acceptance criteria

One of the following:
- [ ] Document in `deploy/README.md` that the test channel must be empty before local validation, OR
- [ ] Add a local env var flag to disable startup reconciliation for smoke tests, OR
- [ ] Log a clear startup reconciliation warning that names the channel and message count being replayed.

**Done when:** a developer running local validation understands what startup reconciliation is doing and why.
