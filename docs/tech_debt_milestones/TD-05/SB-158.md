# SB-158 — Persist last-successful-vault-write metadata

**Branch:** `feature/last-vault-write-metadata`
**Source:** TD-P3-010

---

## Goal

Operators have no durable record of the last successful vault write. This is useful for operational status and diagnosing writer-service degradation.

> **Evaluate after SB-141 (raw vault substrate) and SB-142 (receipt repair tracking).** This item may be partially or fully superseded by those changes.

---

## Acceptance criteria

- [ ] Persist last-successful-vault-write metadata on the capture row or in `system_state`.
- [ ] Include in `uv run secondbrain status` output.
- [ ] If superseded by SB-141 or SB-142, close as duplicate and document the resolution.

**Done when:** `status` output includes the timestamp and capture ID of the last successful vault write.
