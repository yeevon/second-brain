# SB-158 — Persist last-successful-vault-write metadata

**Branch:** `feature/last-vault-write-metadata`
**Source:** TD-P3-010

---

## Status: partially covered — gap remains

TD-04 (SB-142) added receipt sync tracking. SB-141 added the raw vault substrate. Neither fully satisfies this ticket.

Current state in `status` output:

- `last successful vault write:` — shows `derived_note_path` of the most recently updated FILED/INBOX capture. **Present.**
- `last vault write at:` — reads `system_state["last_vault_write_at"]`, written in `complete_filed` / `complete_inbox` / `acknowledge_delivery_filed` / `acknowledge_delivery_inbox`. **Present.**
- `capture_id` of the last successful vault write — **missing.**

The spec requires: timestamp + capture ID + note path. Timestamp and note path exist; capture ID does not.

---

## Implementation

### Step 1 — Write `last_vault_write_capture_id` to `system_state`

**Files:** `src/secondbrain/capture_service.py` — in each of the four methods that already write `last_vault_write_at`:

- `complete_filed` (~line 566)
- `complete_inbox` (~line 612)
- `acknowledge_delivery_filed`
- `acknowledge_delivery_inbox`

Add alongside the existing `last_vault_write_at` write:

```python
self._ledger.set_system_states({
    "last_vault_write_at": datetime.now(UTC).isoformat(),
    "last_vault_write_capture_id": capture_id,
})
```

Use `set_system_states` (batch write) so both keys are updated atomically in the same SQLite write. Do not add a separate `set_system_state` call.

### Step 2 — Surface in status snapshot

**File:** `src/secondbrain/status.py`

Add to `OperationalStatusSnapshot`:

```python
last_vault_write_capture_id: str | None
```

In `_query_snapshot`, alongside the existing `last_vault_write_at` read:

```python
last_vault_write_capture_id = get_state("last_vault_write_capture_id")
```

Pass to the snapshot constructor and include in `format_operational_status`:

```python
f"  last vault write: {_fmt(snapshot.last_vault_write_at)}"
f"    capture: {snapshot.last_vault_write_capture_id or 'none'}"
f"    path:    {snapshot.last_successful_vault_write or 'none'}"
```

(Three lines replacing the current two, keeping all existing fields.)

---

## Acceptance criteria

- [ ] `system_state` key `last_vault_write_capture_id` is written atomically with `last_vault_write_at` on every successful vault write.
- [ ] `status` output shows timestamp, capture ID, and note path for the last successful vault write.
- [ ] If no vault write has occurred, all three show `none` / `None`.
