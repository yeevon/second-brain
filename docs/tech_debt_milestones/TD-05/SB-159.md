# SB-159 — Dockerized MCP profile ownership and safe.directory behavior

**Branch:** `fix/dockerized-mcp-ownership`
**Source:** TD-P3-011 / GitHub #16

---

## Decision: remove Dockerized MCP as a supported path

The `compose.mcp-local.yaml` file already documents the host-process path as preferred:

> "This is NOT the recommended MCP setup. Prefer running brain-mcp as a host process…"

The container mounts the vault read-only (`/opt/vault:ro`) and does not configure `safe.directory` or vault ownership for the `brain-mcp` process UID. Git commands inside the container fail silently or with unhelpful errors. Fixing this requires entrypoint changes, UID mapping, and `safe.directory` injection — all of which replicate complexity already solved in `writer-service`.

The cost of maintaining a correct Dockerized MCP path exceeds the benefit: the host-process path (`uv run brain-mcp` or `brain-mcp` directly) works with all MCP clients without any Docker-specific client configuration, and is already the documented recommendation.

---

## Implementation

### Step 1 — Remove `compose.mcp-local.yaml`

Delete the file. It is only referenced in its own header comment; no other compose file includes it.

### Step 2 — Update documentation

Any reference to the Docker exec MCP setup in `deploy/README.md` (if present) or other docs should be replaced with the host-process setup instructions.

Search for references:

```bash
grep -rn "compose.mcp-local\|mcp-service\|docker.*exec.*brain-mcp" \
    /home/papa-jochy/prj/second-brain/
```

Remove or replace each reference with the canonical host setup:

```
# Host-process MCP setup (all clients):
uv sync
VAULT_PATH=/path/to/vault LEDGER_PATH=/path/to/ledger.sqlite3 uv run brain-mcp

# Claude Code / Claude Desktop config:
{
  "mcpServers": {
    "second-brain": {
      "command": "uv",
      "args": ["run", "--project", "/path/to/second-brain", "brain-mcp"],
      "env": {
        "VAULT_PATH": "/path/to/my-vault",
        "LEDGER_PATH": "/path/to/.runtime/ledger.sqlite3"
      }
    }
  }
}
```

### Step 3 — Architecture test

Add one test to `tests/architecture/test_container_config.py` (or `test_compose_orchestration.py`):

```python
def test_mcp_local_compose_file_does_not_exist():
    """Dockerized MCP is explicitly unsupported; the file must be absent."""
    assert not (ROOT / "compose.mcp-local.yaml").exists(), (
        "compose.mcp-local.yaml was removed as an unsupported path. "
        "Use the host-process brain-mcp instead."
    )
```

This test fails if someone re-adds the file, making the removal durable.
