# SB-151 — Stronger n8n readiness signal or documented sufficiency

**Branch:** `fix/n8n-readiness-signal`
**Source:** TD-P3-003 / GitHub #8

---

## Goal

The current n8n readiness signal worked in practice but may be fragile across n8n version upgrades. Either improve it or explicitly document why it is sufficient.

---

## Acceptance criteria

One of the following:
- [ ] Identify and use a stronger readiness signal that is stable across n8n versions, OR
- [ ] Extend init retry behavior around import/activation/webhook registration to tolerate transient not-ready states, OR
- [ ] Document in `deploy/README.md` why the current readiness signal is sufficient and what version range it covers.

**Done when:** n8n readiness behavior is either more robust or explicitly documented as intentional.
