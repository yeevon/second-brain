# SB-151 — Stronger n8n readiness signal or documented sufficiency

**Branch:** `fix/n8n-readiness-signal`
**Source:** TD-P3-003 / GitHub #8

---

## Goal

The current n8n readiness signal worked in practice but may be fragile across n8n version upgrades. Make the init script tolerate transient not-ready states rather than relying on a single-shot healthcheck pass.

---

## Decision

**Selected path: extend init retry behavior around import/activation/webhook registration.**

Rationale: the healthcheck in `compose.override.yaml` probes `/rest/login` with an empty body and expects HTTP 400 (bad request = login endpoint reachable). This is stable across n8n 2.x. The fragility is not the healthcheck itself but the window between the healthcheck passing and the REST API being fully operational for workflow imports. Adding per-step retries in `local-n8n-init.py` is safer than relying on timing and does not require undocumented n8n internals.

---

## Implementation

**File:** `deploy/local-n8n-init.py`

Add a `_retry` helper that wraps any callable with configurable attempts and a fixed delay:

```python
def _retry(fn, *, attempts=5, delay_s=3, label=""):
    for attempt in range(1, attempts + 1):
        try:
            return fn()
        except Exception as exc:
            if attempt == attempts:
                raise
            print(f"  {label} attempt {attempt} failed ({exc}), retrying in {delay_s}s…")
            time.sleep(delay_s)
```

Wrap the three steps that are most sensitive to n8n not being fully ready:

1. **Credential creation** — already guarded by `_find_credential`, but wrap the creation call.
2. **Workflow import/update** — wrap `_import_or_update_workflow` per workflow.
3. **Workflow activation** — wrap `_activate_workflow`.
4. **Webhook verification** — wrap `_verify_webhook`.

Default: `attempts=5`, `delay_s=3` (15 s total window per step — enough for n8n to finish indexing without adding significant startup time in the happy path).

These defaults can be overridden via env vars `N8N_INIT_RETRY_ATTEMPTS` and `N8N_INIT_RETRY_DELAY_S` for environments where n8n starts more slowly.

**No changes to the healthcheck condition.** The `POST /rest/login → 400` probe is correct and stable for n8n 2.x.

---

## Acceptance criteria

- [ ] `local-n8n-init.py` wraps workflow import, activation, and webhook verification with per-step retries.
- [ ] Default is 5 attempts with 3 s delay.
- [ ] `N8N_INIT_RETRY_ATTEMPTS` and `N8N_INIT_RETRY_DELAY_S` env vars override the defaults.
- [ ] Each retry logs attempt number and error reason.
- [ ] Final failure after exhausting retries still exits non-zero.

## Test location

`tests/unit/test_td05.py`

**Assertion:** test that `_retry` raises on exhaustion and logs intermediate attempts, and that it returns the value on eventual success.

```python
def test_sb151_retry_raises_after_exhaustion():
    calls = [0]
    def always_fail():
        calls[0] += 1
        raise RuntimeError("not ready")
    with pytest.raises(RuntimeError, match="not ready"):
        _retry(always_fail, attempts=3, delay_s=0)
    assert calls[0] == 3

def test_sb151_retry_returns_on_eventual_success():
    calls = [0]
    def fail_twice():
        calls[0] += 1
        if calls[0] < 3:
            raise RuntimeError("not ready")
        return "ok"
    result = _retry(fail_twice, attempts=5, delay_s=0)
    assert result == "ok"
    assert calls[0] == 3
```

**Done when:** init retries transiently failing steps and exits non-zero only after all retries are exhausted.
