# SB-150 — Stale local n8n owner mismatch diagnostics

**Branch:** `fix/n8n-init-owner-mismatch`
**Source:** TD-P3-002 / GitHub #8

---

## Goal

When `deploy/local-n8n-init.py` gets a 401 during login due to an owner mismatch (stale volume, changed credentials), the error message gives no actionable guidance.

---

## Root cause

`login()` in `deploy/local-n8n-init.py` calls `_api("POST", "/rest/login", ...)`. The `_api` helper raises a bare `RuntimeError` with the HTTP status and raw response body when a non-`ok_status` code is returned. A 401 from `/rest/login` is a known, recoverable local setup situation (stale n8n data volume or changed `N8N_LOCAL_PASSWORD`), but the current error message does not distinguish it from unexpected failures.

---

## Implementation

**File:** `deploy/local-n8n-init.py` — `login()` function

Wrap the `_api` call with an explicit 401 check. The `_api` helper raises `RuntimeError` for unexpected statuses; instead, call with `ok_statuses=None` to get the raw status back and handle it:

```python
def login():
    status, body = _api("POST", "/rest/login", {
        "emailOrLdapLoginId": LOCAL_EMAIL,
        "password": LOCAL_PASSWORD,
    }, ok_statuses=None)
    if status == 401:
        print(
            "\nERROR: n8n login returned 401 — owner account credentials do not match.\n"
            "\nLikely causes:\n"
            "  1. Stale n8n data volume from a previous session with different credentials.\n"
            "     Fix: docker compose down && docker volume rm second-brain-local-n8n-data\n"
            "          then re-run docker compose up -d\n"
            "  2. N8N_LOCAL_EMAIL or N8N_LOCAL_PASSWORD changed since the volume was created.\n"
            "     Fix: align the env vars with the credentials stored in the volume,\n"
            "          or wipe the volume as above.\n"
            "  3. Owner was never created (setup step skipped).\n"
            "     Fix: open http://localhost:5678 and complete the owner setup in the UI.\n"
            "\nDo not include passwords in bug reports.",
            file=sys.stderr,
        )
        sys.exit(1)
    if status not in (200, 201):
        raise RuntimeError(
            f"POST /rest/login → HTTP {status}: {json.dumps(body)[:300]}"
        )
    data = _unwrap(body)
    if not isinstance(data, dict) or "email" not in data:
        raise RuntimeError(f"Login response did not contain user object: {body}")
    print(f"  Logged in as {data['email']}")
```

**Exit behavior:** `sys.exit(1)` — non-zero exit. The `local-n8n-init` Compose service will report `service_completed_unsuccessfully`, blocking `capture-service` startup (per `depends_on: condition: service_completed_successfully`). This is correct — a broken init should stop the stack, not silently proceed.

**Which 401s get the message:** only the `/rest/login` path. The `setup_owner` path already returns 400 (owner exists) or 200 (created). No other calls are expected to 401 under normal operation.

**No secrets in output:** the message references `N8N_LOCAL_EMAIL`/`N8N_LOCAL_PASSWORD` by name only; the values of `LOCAL_EMAIL` and `LOCAL_PASSWORD` must never appear in the printed text.

---

## Acceptance criteria

- [ ] 401 from `/rest/login` prints a remediation message naming stale volume, changed credentials, and manual UI creation as resolution paths.
- [ ] The printed message does not include the value of `LOCAL_EMAIL` or `LOCAL_PASSWORD`.
- [ ] Exit code is non-zero (`sys.exit(1)`) on 401.
- [ ] Non-401 unexpected statuses still raise `RuntimeError` as before.

## Test location

`tests/unit/test_td05.py`

**Assertion:** import `login` from `deploy/local-n8n-init.py` (or test via subprocess with a mock server). Since the script uses module-level globals, the simplest test is a unit test that patches `_api` to return `(401, {})` and asserts that `SystemExit` is raised with code 1 and that `sys.stderr` output contains "stale" and does not contain the password value.

```python
def test_sb150_login_401_exits_with_remediation(monkeypatch, capsys):
    import importlib, sys
    monkeypatch.setenv("N8N_LOCAL_PASSWORD", "s3cr3t")
    monkeypatch.setenv("N8N_INTAKE_WEBHOOK_TOKEN", "tok")
    monkeypatch.setenv("CAPTURE_SERVICE_INTERNAL_TOKEN", "tok")
    monkeypatch.setenv("GEMINI_API_KEY", "key")
    spec = importlib.util.spec_from_file_location("n8n_init", "deploy/local-n8n-init.py")
    mod = importlib.util.module_from_spec(spec)
    spec.loader.exec_module(mod)
    monkeypatch.setattr(mod, "_api", lambda *a, **kw: (401, {}))
    with pytest.raises(SystemExit) as exc:
        mod.login()
    assert exc.value.code == 1
    captured = capsys.readouterr()
    assert "stale" in captured.err.lower()
    assert "s3cr3t" not in captured.err
```

**Done when:** a 401 from login prints clear guidance and exits non-zero without leaking credentials.
