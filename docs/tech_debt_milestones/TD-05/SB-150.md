# SB-150 — Stale local n8n owner mismatch diagnostics

**Branch:** `fix/n8n-init-owner-mismatch`
**Source:** TD-P3-002 / GitHub #8

---

## Goal

When `deploy/local-n8n-init.py` gets a 401 due to an owner mismatch (stale volume, changed credentials), the error message gives no actionable guidance. Operators spend unnecessary time diagnosing what is a well-known local setup issue.

---

## Acceptance criteria

- [ ] Catch login 401 in `deploy/local-n8n-init.py`.
- [ ] Print a remediation message that includes:
  - Stale n8n data volume as a likely cause
  - Changed local email/password as a likely cause
  - Manual owner creation in UI as a resolution path
- [ ] Do not print the password or any secret values.
- [ ] Add a targeted test for the 401 path.

**Done when:** a 401 owner mismatch prints a clear actionable message without leaking credentials.
