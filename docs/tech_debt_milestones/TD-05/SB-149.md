# SB-149 — Startup reconciliation deferred-capture log clarity

**Branch:** `fix/startup-reconciliation-log-message`
**Source:** TD-P3-001 / GitHub #5

---

## Goal

Startup/history reconciliation logs `downstream processing disabled` for intentionally deferred historical messages. This looks like an error when it is normal behavior.

---

## Acceptance criteria

- [ ] Historical/startup messages log a distinct reason: `historical_reconciliation_deferred`.
- [ ] `downstream processing disabled` is used only when downstream processing is actually disabled at the service level.
- [ ] Logs clearly show whether a capture was queued for n8n delivery or intentionally deferred.

**Done when:** the log reason for a deferred historical message is unambiguous and does not look like a downstream error.
