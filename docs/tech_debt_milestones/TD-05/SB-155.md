# SB-155 — Compose orchestration regression tests

**Branch:** `feature/compose-orchestration-tests`
**Source:** TD-P3-007 / GitHub #8

---

## Goal

Local Compose service ordering is not protected by tests. A future change to `compose.override.yaml` could silently break the startup sequence.

---

## Implementation

**Test file:** `tests/architecture/test_compose_orchestration.py` (new file)

**Compose file read:** `compose.override.yaml` (the local stack — this is where `depends_on` is defined; `compose.yaml` only defines the capture-service base without dependencies)

Use the existing pattern from `test_container_config.py`:

```python
import yaml
from pathlib import Path

ROOT = Path(__file__).parent.parent.parent

def load_override():
    return yaml.safe_load((ROOT / "compose.override.yaml").read_text())
```

**Assertions:**

1. `n8n` service defines a `healthcheck`.
2. `local-n8n-init` has `depends_on.n8n.condition == "service_healthy"`.
3. `capture-service` has `depends_on.local-n8n-init.condition == "service_completed_successfully"`.
4. `capture-service` has `depends_on.writer-service.condition == "service_healthy"`.
5. `writer-service` has `depends_on.local-vault-init.condition == "service_completed_successfully"`.

These five assertions cover the full startup chain:

```
local-vault-init ──(completed)──▶ writer-service
n8n ──(healthy)──▶ local-n8n-init ──(completed)──▶ capture-service
writer-service ──(healthy)──▶ capture-service
```

---

## Acceptance criteria

- [ ] `tests/architecture/test_compose_orchestration.py` exists with the five dependency assertions above.
- [ ] Tests pass against the current `compose.override.yaml`.
- [ ] Any removal or weakening of a `depends_on` condition causes a test failure.

---

## Test structure

```python
def test_n8n_defines_healthcheck():
    svc = load_override()["services"]["n8n"]
    assert "healthcheck" in svc

def test_local_n8n_init_depends_on_n8n_healthy():
    deps = load_override()["services"]["local-n8n-init"]["depends_on"]
    assert deps["n8n"]["condition"] == "service_healthy"

def test_capture_service_depends_on_n8n_init_completed():
    deps = load_override()["services"]["capture-service"]["depends_on"]
    assert deps["local-n8n-init"]["condition"] == "service_completed_successfully"

def test_capture_service_depends_on_writer_service_healthy():
    deps = load_override()["services"]["capture-service"]["depends_on"]
    assert deps["writer-service"]["condition"] == "service_healthy"

def test_writer_service_depends_on_vault_init_completed():
    deps = load_override()["services"]["writer-service"]["depends_on"]
    assert deps["local-vault-init"]["condition"] == "service_completed_successfully"
```

**Done when:** the five tests pass and any future dependency removal breaks the suite.
