# SB-155 — Compose orchestration regression tests

**Branch:** `feature/compose-orchestration-tests`
**Source:** TD-P3-007 / GitHub #8

---

## Goal

Local Compose service ordering is not protected by tests. A future change to `docker-compose.yml` could silently break the startup sequence, which requires: n8n healthy before init, init complete before capture-service, writer-service healthy before capture-service.

---

## Acceptance criteria

- [ ] Tests assert Compose service dependency ordering:
  - n8n healthcheck readiness path is defined
  - `local-n8n-init` depends on n8n healthy
  - `capture-service` depends on `local-n8n-init` completing successfully
  - `capture-service` depends on `writer-service` healthy

**Done when:** any breakage of the Compose startup ordering is caught by the architecture test suite.
