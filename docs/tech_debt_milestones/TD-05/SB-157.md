# SB-157 — Centralize structured application logging

**Branch:** `refactor/centralize-structured-logging`
**Source:** TD-P3-009

---

## Goal

Structured logging is currently spread across call sites. Centralizing it would make it easier to enforce the safe-metadata-only invariant and add fields consistently.

> **Only prioritize if nearby logging work makes it cheap.** SB-148 (SQLite contention instrumentation) and SB-137 (background task liveness) both touch logging — evaluate whether centralizing is worth the churn after those are complete.

---

## Acceptance criteria

- [ ] A single entry point or wrapper for all structured log events.
- [ ] Enforcement point for safe-metadata-only policy (no raw exception bodies, no secrets).
- [ ] Existing `log_metadata` call sites migrated to the centralized wrapper.
- [ ] All existing log event tests continue to pass.

**Done when:** all structured log events flow through one centralized path with a single enforcement point for the safe-metadata policy.
