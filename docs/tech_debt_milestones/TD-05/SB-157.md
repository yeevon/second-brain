# SB-157 — Centralize structured application logging

**Branch:** `refactor/centralize-structured-logging`
**Source:** TD-P3-009

---

## Goal

Add a single enforcement point inside `log_metadata` that rejects or redacts unsafe field values before they reach stdout. Do not migrate all call sites — the existing `log_metadata` API is already the single entry point.

---

## Scope

111 call sites in `src/secondbrain/` already use `log_metadata`. The function is already the sole entry point for structured logs (no bare `print()` for service-level events). The missing piece is enforcement: nothing prevents a caller from accidentally passing a raw exception message or secret value as a field.

**What this ticket does:**

1. Add field-value validation inside `log_metadata` in `src/secondbrain/observability.py`.
2. Reject values that look like raw exception bodies or Python tracebacks.
3. Add tests proving the enforcement works.

**What this ticket does not do:**

- Migrate call sites (they already use `log_metadata`).
- Change the function signature.
- Touch writer-service logging (separate package).

---

## Implementation

**File:** `src/secondbrain/observability.py`

Add a `_is_safe_field_value(value)` guard and call it for every value in `**fields` before writing to stdout:

```python
import re as _re

_TRACEBACK_PATTERN = _re.compile(r"Traceback \(most recent call last\)", _re.IGNORECASE)
_EXCEPTION_BODY_PATTERN = _re.compile(r"[A-Za-z]+Error: .{40,}", _re.IGNORECASE)

_MAX_FIELD_VALUE_LEN = 500

def _safe_field_value(key: str, value: object) -> object:
    """Truncate or redact unsafe field values before logging."""
    if not isinstance(value, str):
        return value
    if len(value) > _MAX_FIELD_VALUE_LEN:
        return value[:_MAX_FIELD_VALUE_LEN] + "…[truncated]"
    if _TRACEBACK_PATTERN.search(value):
        return f"[traceback redacted — use error_type instead, key={key!r}]"
    if _EXCEPTION_BODY_PATTERN.search(value):
        return f"[exception body redacted — use error_type instead, key={key!r}]"
    return value
```

Apply inside `log_metadata`:

```python
def log_metadata(event_name: str, **fields) -> None:
    safe_fields = {k: _safe_field_value(k, v) for k, v in fields.items()}
    record = {"event": event_name, "level": "INFO", ..., **safe_fields}
    print(json.dumps(record), flush=True)
```

**Invariants preserved:**

- `error_type=type(exc).__name__` (the canonical pattern) always passes — it is a short class name with no body.
- Numeric, boolean, and `None` values pass through unchanged.
- Short strings pass through unchanged.

## Acceptance criteria

- [ ] `log_metadata` redacts field values matching a Python traceback pattern.
- [ ] `log_metadata` redacts field values matching a long exception body pattern.
- [ ] `log_metadata` truncates field values longer than 500 characters.
- [ ] `error_type=type(exc).__name__` is not affected (short class name always passes).
- [ ] All existing log event tests continue to pass.

## Test location

`tests/unit/test_td05.py`

```python
def test_sb157_traceback_in_field_is_redacted(capsys):
    log_metadata("test_event", detail="Traceback (most recent call last):\n  File...")
    out = capsys.readouterr().out
    assert "Traceback" not in out
    assert "redacted" in out

def test_sb157_exception_body_in_field_is_redacted(capsys):
    log_metadata("test_event", detail="ValueError: something went badly wrong " + "x" * 60)
    out = capsys.readouterr().out
    assert "something went badly wrong" not in out

def test_sb157_error_type_classname_passes_through(capsys):
    log_metadata("test_event", error_type="ValueError")
    out = capsys.readouterr().out
    assert "ValueError" in out

def test_sb157_long_string_is_truncated(capsys):
    log_metadata("test_event", detail="x" * 600)
    out = capsys.readouterr().out
    assert "truncated" in out
    assert len(json.loads(out.strip())["detail"]) < 520
```

**Done when:** raw exception bodies and tracebacks cannot reach stdout through `log_metadata`, and all 111 existing call sites continue to work without modification.
