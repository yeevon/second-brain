# SB-152 — Deterministic local classifier mode for smoke tests

**Branch:** `feature/deterministic-classifier-mode`
**Source:** TD-P3-004 / GitHub #8

---

## Goal

Local smoke tests always burn Gemini API tokens and depend on model behavior. A deterministic classifier mode makes local validation faster, cheaper, and reproducible.

---

## Contract

### Env var

`CLASSIFIER_MODE` — values: `gemini` | `deterministic`

- Default: `gemini` (no behavior change when unset or set to `gemini`)
- `deterministic` — skips all Gemini HTTP calls and returns a fixed classification

### Preflight behavior

When `CLASSIFIER_MODE=deterministic`:
- Do **not** validate or require `GEMINI_API_KEY`
- Log `classifier_mode_deterministic` at startup (event name, no fields needed beyond mode)

When `CLASSIFIER_MODE=gemini` (or unset):
- Existing Gemini key validation behavior is unchanged

### Deterministic classification output (hardcoded, not env-configurable)

```python
DETERMINISTIC_CLASSIFICATION = {
    "folder": "inbox",
    "project": None,
    "note_type": "note",
    "title": "Deterministic Test Capture",
    "tags": ["smoke-test"],
    "body": "This note was filed by the deterministic classifier.",
    "actions": [],
    "needs_clarification": False,
    "clarifying_question": None,
    "confidence": 1.0,
    "note_date": None,
}
```

This is a valid `Classification` per the writer-service schema. `folder=inbox` ensures the capture ends up in `INBOX` (not `FILED`) so it cannot be confused with a real classified note.

### Attachment-only captures

When `raw_text == ""` and `has_attachments == True`, the deterministic classifier returns the same fixed output with `folder="inbox"` — same as the Gemini path (attachment-only captures are classified by n8n, not by the local worker). No special-casing needed.

### Where classification happens

`local-full` mode only. The deterministic classifier replaces the Gemini call inside `src/secondbrain/classifier.py` (or wherever `classify_text` / the Gemini call is made). In `capture-only` mode, the classifier is not invoked by capture-service at all (classification is done by n8n); `CLASSIFIER_MODE` is ignored in that mode.

**File:** `src/secondbrain/classifier.py` — branch on `CLASSIFIER_MODE` env var before calling Gemini. Do not pass the env var as a constructor argument; read it once at call time (or at module load, whichever is simpler).

### Tests using deterministic mode

Existing unit and integration tests that exercise the classification path should set `CLASSIFIER_MODE=deterministic` via `monkeypatch.setenv` or by passing a `FakeClassifier` (which is already how they work). No changes to existing test fakes.

The smoke test (`deploy/test-writer-service.sh` and similar) should document that `CLASSIFIER_MODE=deterministic` may be set to skip Gemini calls. The real Gemini path remains opt-in for manual validation.

---

## Acceptance criteria

- [ ] `CLASSIFIER_MODE=deterministic` causes the classifier to return `DETERMINISTIC_CLASSIFICATION` without any Gemini HTTP call.
- [ ] `CLASSIFIER_MODE=gemini` (or unset) is unchanged.
- [ ] Gemini API key is not required when `CLASSIFIER_MODE=deterministic`.
- [ ] Startup logs `classifier_mode_deterministic` event when deterministic mode is active.
- [ ] `DETERMINISTIC_CLASSIFICATION` is a valid `Classification` (folder=inbox, passes schema validation).
- [ ] `capture-only` mode ignores `CLASSIFIER_MODE`.

## Test location

`tests/unit/test_td05.py`

```python
def test_sb152_deterministic_mode_returns_fixed_classification(monkeypatch):
    monkeypatch.setenv("CLASSIFIER_MODE", "deterministic")
    result = classify_text("anything")  # or however the function is called
    assert result["folder"] == "inbox"
    assert result["confidence"] == 1.0
    assert "smoke-test" in result["tags"]

def test_sb152_deterministic_mode_does_not_require_gemini_key(monkeypatch):
    monkeypatch.setenv("CLASSIFIER_MODE", "deterministic")
    monkeypatch.delenv("GEMINI_API_KEY", raising=False)
    # must not raise
    result = classify_text("anything")
    assert result is not None

    ...
```
