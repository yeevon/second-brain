# SB-145 — Explicit invalid-classifier fallback in n8n Intake

**Branch:** `feature/invalid-classifier-fallback`
**Source:** TD-P2-004 / GitHub #16

---

## Goal

When the Gemini classifier returns `valid=false` or `route=null`, the capture currently waits for stale-lease recovery to pick it up. This is unnecessary latency and obscures the root cause. Invalid classifier output must route immediately to a defined failure path.

---

## Acceptance criteria

- [ ] `valid=false` or `route=null` in classifier output immediately routes to retry or terminal failure — no stale-lease wait.
- [ ] The route decision is logged with a safe event type.
- [ ] Add n8n workflow test for invalid classifier output.
- [ ] Tests cover:
  - `valid=false` — immediate failure route
  - `route=null` — immediate failure route
  - Valid classifier output — normal processing path unchanged

**Done when:** schema-invalid Gemini output is handled immediately without waiting for stale-lease recovery.
