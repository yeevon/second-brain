# SB-146 — Writer-service classification schema and Markdown renderer metadata sync

**Branch:** `feature/writer-service-metadata-sync`
**Source:** TD-P2-005 / GitHub #16

---

## Goal

Writer-service models and the Markdown renderer must accept and preserve the same metadata fields used by capture-service. Without this, notes lose `note_date`, action `due`, `priority`, and `project` in transit through writer-service.

---

## Current gap

**capture-service `models.py`** has:
- `Classification.note_date: str | None`
- `ClassifiedAction.due: str | None`
- `ClassifiedAction.priority: ActionPriority | None`
- `ClassifiedAction.project: str | None`

**writer-service `api_models.py`** is missing all four. `Classification` has no `note_date`; `ClassifiedAction` only has `text` and `status`.

The Markdown renderer in `writer.py` (`_build_frontmatter_and_body`) does not write `note_date`, `action.due`, `action.priority`, or `action.project` to frontmatter. The brief scanner (`brief.py`) already reads `due`, `priority`, `project` from frontmatter actions — so the gap is in the writer, not the scanner.

---

## Schema changes (writer-service `api_models.py`)

```python
ActionPriority = Literal["high", "medium", "low"]

class ClassifiedAction(BaseModel):
    model_config = ConfigDict(extra="forbid")

    text: str = Field(min_length=1)
    status: ActionStatus
    due: str | None = None        # ISO date "YYYY-MM-DD"
    priority: ActionPriority | None = None
    project: str | None = None    # project slug

class Classification(BaseModel):
    # ... existing fields ...
    note_date: str | None = None  # ISO date for birthday/event/reminder notes
    # ... rest unchanged ...
```

All new fields have `None` defaults — requests without these fields continue to work. `extra="forbid"` is preserved.

---

## Renderer changes (writer-service `writer.py`)

In `_build_frontmatter_and_body`:

1. After `note_type:`, add:
   ```
   if classification.note_date:
       lines.append(f"note_date: {yaml_scalar(classification.note_date)}")
   ```

2. In the actions loop, add `due`, `priority`, `project` after `status:`:
   ```
   lines.append(f"  - text: {yaml_scalar(action.text)}")
   lines.append(f"    status: {yaml_scalar(action.status)}")
   if action.due:
       lines.append(f"    due: {yaml_scalar(action.due)}")
   if action.priority:
       lines.append(f"    priority: {yaml_scalar(action.priority)}")
   if action.project:
       lines.append(f"    project: {yaml_scalar(sanitize_slug(action.project))}")
   ```

Fields are omitted (not written as `null`) when absent — the scanner treats missing fields as `None`.

```yaml
note_type: "birthday"

actions:

    status: open
    due: "2026-08-14"
    priority: high
    project: personal
```

## Brief test coverage

Writer-service brief tests (`writer-service/tests/`) must cover notes filed through writer-service that include these fields:
- Birthday note with `note_date` → appears in `birthdays` section of daily brief
- Event/reminder with `note_date` → appears in `coming_up`
- Action with `due` and `priority` → appears in `still_open` with correct `due` and `priority` values
