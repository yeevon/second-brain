# SB-146 — Writer-service classification schema and Markdown renderer metadata sync

**Branch:** `feature/writer-service-metadata-sync`
**Source:** TD-P2-005 / GitHub #16

---

## Goal

Writer-service models and the Markdown renderer must accept and preserve the same metadata fields used by capture-service and n8n. Without this, briefs and structured notes lose metadata in transit.

---

## Acceptance criteria

- [ ] Writer-service models accept the same metadata fields as capture-service and n8n schemas.
- [ ] Renderer preserves and surfaces:
  - `note_date`
  - Action `due`
  - Action `priority`
  - Action `project`
- [ ] Daily/Weekly brief tests cover notes filed through writer-service:
  - Birthday notes
  - Event notes
  - Reminder notes
  - Due-date action notes

**Done when:** metadata fields are preserved end-to-end through writer-service and appear correctly in generated briefs.
