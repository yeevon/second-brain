# SB-147 — Fallback weekly scan explicit completion rules

**Branch:** `feature/weekly-scan-completion-rules`
**Source:** TD-P2-006 / GitHub #16

---

## Goal

The fallback weekly scan can incorrectly treat ordinary notes and tasks as completed accomplishments. Only explicitly completed notes and tasks should count.

---

## Current behavior

In `brief.py:scan_weekly_brief()`:

**Accomplished notes:** already correct — only `note_type in ("done", "fix")` are added to `accomplished`. No change needed.

**Completed tasks:** line 272 accepts `action.get("status") in ("done", "completed")`. The value `"completed"` is not part of the canonical `ActionStatus` enum (`"open" | "done"` in both capture-service and writer-service `api_models.py`). This was a defensive check against inconsistent data but is now ambiguous — it implies `"completed"` is a valid state when it isn't.

---

## Decision

**`action.status == "done"` is the only canonical completed state for tasks.**

- `_parse_actions` already parses `status:` from frontmatter as a raw string.
- The weekly scan must accept only `"done"` as completed.
- `"completed"` is not a valid `ActionStatus` value and must not be treated as such.
- Markdown checkbox state (`[x]`) is **not** a source of truth for completion — the structured `status:` field is canonical. Checkboxes in the body are rendering artifacts only.

---

## Change required

In `brief.py`, line 272, change:

```python
# Before
if action.get("status") in ("done", "completed") and created_this_week:
```

to:

```python
# After
if action.get("status") == "done" and created_this_week:
```

No other changes to `scan_weekly_brief` logic.

---

## Acceptance criteria

- [ ] `note_type: done` — counted as accomplished in `scan_weekly_brief`.
- [ ] `note_type: fix` — counted as accomplished.
- [ ] All other note types — not counted as accomplished (unchanged behavior).
- [ ] Action with `status: done` created this week — counted as completed task.
- [ ] Action with `status: open` — not counted as completed task.
- [ ] Action with `status: completed` (legacy/invalid value) — **not** counted as completed task.
- [ ] Markdown checkbox state has no effect on completion counting.
- [ ] Tests cover all five cases above.

**Done when:** the weekly scan only surfaces genuinely completed work and the rules are covered by tests.
