# SB-144 — Handle attachment-only captures before text security screening

**Branch:** `feature/attachment-only-capture-path`
**Source:** TD-P2-003 / GitHub #16

---

## Goal

Attachment-only Discord messages produce empty `raw_text`. Text security screening may fail or reject these captures incorrectly. The path for attachment-only captures must be safe and well-defined.

---

## Acceptance criteria

- [ ] Attachment-only captures route before text screening, OR text screening accepts empty `raw_text` safely.
- [ ] Add workflow/architecture test for attachment-only path.
- [ ] Operator-visible outcome is clear — capture is accepted and routed correctly.
- [ ] Tests cover:
  - Attachment-only message — accepted and routed
  - Empty `raw_text` does not trigger a secret-screening rejection

**Done when:** an attachment-only Discord message is captured successfully and the path is covered by an architecture or workflow test.
